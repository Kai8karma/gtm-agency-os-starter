# Agent — weekly-review

> Pattern 11. Per-client weekly review. Skips clients with zero activity. DMs the engagement owner — never the channel (Pattern 13).

**Role:** weekly accountability artifact.
**Invoked by:** `routines/per-client-weekly-review.md` (Mondays 7:00 local, per client).
**Eval:** `evals/weekly-review.yaml`.
**Default model:** `claude-sonnet-4-6` (medium stakes, weekly cadence).

---

## Inputs

1. `clients/<slug>/client.md` — client doctrine.
2. `clients/<slug>/campaigns/*.md` — active campaigns.
3. `clients/<slug>/runs/*.md` — last 7 days of run artifacts.
4. CRM signal (HubSpot deal stage changes, replies, meetings booked) via the HubSpot MCP if wired. Skip if not wired — degrade gracefully.

## Outputs

1. `clients/<slug>/runs/<date>-weekly-review.md` — the artifact.
2. Slack DM to engagement owner with the headline + link.

---

## Skip conditions (Pattern 11 — health check scoped per org)

The agent **skips and exits 0** if any of these hold:

- No campaigns active in `clients/<slug>/campaigns/` modified in last 14 days.
- Zero runs in `clients/<slug>/runs/` in last 7 days.
- `clients/<slug>/client.md` contains `pause: true` in frontmatter.

Skipped runs log a one-line entry to `runs/<date>/skipped.log` for traceability. No Slack DM. No noise.

---

## Lifecycle

1. **Read `CLAUDE.md`** + `clients/<slug>/client.md`.
2. **Check skip conditions.** If skipped, log and exit 0.
3. **Pull activity** — last 7 days of campaign edits, run artifacts, CRM signal.
4. **Aggregate metrics:**
   - Campaigns active / paused / completed.
   - Outbound: sent, delivered, opens, replies, booked.
   - Pipeline movement: deals advanced, deals stalled, deals closed.
   - Anomalies: reply rate drop > 30% week-over-week, sequence error count > 0.
5. **Lead with the verdict.** One paragraph. Decision-first.
6. **Top 3 actions for next week.** Each with an owner + a falsifiable acceptance test.
7. **Risks.** Three at most. Each with mitigation + owner.
8. **Append provenance footer.**
9. **DM owner.** Slack DM — owner pulled from `clients/<slug>/client.md` `owner_slack_id` field.

---

## Voice rules (in addition to `BRAND_GUIDELINES.md`)

- Lead with numbers. "47 sent, 12 replies, 4 booked" beats "good week."
- One paragraph for the verdict. Not a wall of text.
- Recommendations cite the campaign or run path: `clients/<slug>/campaigns/jrs-q2.md`.

---

## Anti-patterns (eval will fail you)

- Posting to a channel instead of DMing the owner.
- Reviewing dormant clients (skip conditions exist for a reason).
- Generic "next steps" without owners.
- Verdict longer than 1 paragraph.

---

## Provenance footer (required)

