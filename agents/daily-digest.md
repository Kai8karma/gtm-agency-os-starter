# Agent — daily-digest

> Pattern 10 (closed-loop) + Pattern 13 (DM the human, not the channel). The heartbeat of the OS — every owner gets a 7am DM with what slipped, what shipped, what's due.

**Role:** daily personal accountability brief, per owner, per agency.
**Invoked by:** `routines/daily-digest.md` (cron: weekdays 7:00 local).
**Eval:** `evals/daily-digest.yaml`.
**Default model:** `claude-haiku-4-5` (daily, low-stakes-per-run, high volume).

---

## Inputs

1. Open tasks from the task store (Notion DB or sqlite — see `routines/task-cron.md`).
2. Yesterday's run artifacts across all clients (`clients/*/runs/<yesterday>.md`).
3. Calendar (Google Calendar MCP if wired) for today's meetings.
4. Inbound triage queue (`Respond`-tier items that didn't get a reply yesterday).

## Outputs

A single Slack DM per owner, formatted as:

```
*Today — <YYYY-MM-DD>*

✓ Shipped yesterday (3)
  • <one-line>
  • <one-line>
  • <one-line>

⚠ Slipped (2) — DM you because you own them
  • <task> — due <date>, <N> days late
  • <task> — due <date>, <N> days late

⏳ Due today (4)
  • <task> — owner: <you>
  • <task> — owner: <you>

📅 Meetings (2)
  • 10:00 — <title> (<attendees>)
  • 14:30 — <title> (<attendees>)

🚨 Respond-tier inbound (1)
  • <prospect> — <client> — quoted: "<phrase>"
    → draft: clients/<slug>/runs/<date>-triage-<thread>.md

Generated by agents/daily-digest.md at 07:00:12. Eval: pass (8.7/10).
```

---

## Lifecycle

1. **Per owner**, pull their open + slipped + due-today tasks.
2. **Pull yesterday's shipped** — closed tasks from `runs/<yesterday>` matching this owner.
3. **Pull calendar** for today (if connector wired). Skip section if not.
4. **Pull Respond-tier triage** that's older than 24h with no reply.
5. **Compose** the DM. Hard cap 30 lines. Truncate with "+ N more — see <link>" if longer.
6. **Send via Slack DM** to the owner's `slack_id` (resolved from `clients/*/client.md` `owner_slack_id` or agency roster).
7. **Log** to `runs/<date>/daily-digest-<owner>.md`.

---

## Voice rules

- Lead with shipped. Wins first. Slipped second. Due today last.
- Slipped items always quote the days late. Pressure = useful.
- Never use a channel post. DM only (Pattern 13).
- Empty sections collapse — no "Nothing slipped today!" filler.

---

## Anti-patterns (eval will fail you)

- Posting to a channel.
- More than 30 lines.
- Including someone else's tasks.
- "Nothing to report" (just don't send the DM that day).
- Missing the provenance footer.

---

## Provenance footer (required)

```
Generated by agents/daily-digest.md at <ISO timestamp>.
Eval: <pass/fail> (<score>/10).
Run: runs/<date>/daily-digest-<owner>.md
```
